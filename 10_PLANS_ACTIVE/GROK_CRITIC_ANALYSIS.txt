Layer 3 — THE DEEPEST ROOT CAUSE (still present)
Value-target aliasing from stochastic hidden hazard timing + obs/reward signal desynchronization.
Break it down rigorously:

Stochastic hidden timing (the dominant aliasing source)
Hazard injection: hazard_step ~ Uniform(150, 350) inside a 500-step episode.
HAZARD_PROBABILITY = 0.5 (Runs 015/016) or 1.0 (Run 014).
For any state before the sampled hazard_step (steps 0–149 in every episode), the observation is identical across episodes.
Yet the return distribution is wildly different:
Hazard early (step 150) → ~350 steps of PENALTY_IGNORING_HAZARD (–5 × 350 = –1 750).
Hazard late (step 350) → ~150 steps of penalty (–750).
No hazard at all (prob 0.5) → calm cruising return ~+200–300.

The critic is asked to predict a single value for states that have irreducible variance in future return. PPO’s value loss explodes; explained_variance collapses to 0.
Run 011 escaped this only because setSpeed(0) created an instant, massive spike (–13.9 m/s²) that made the onset state instantly distinguishable. Gradual slowDown removed the spike → aliasing returned.

Obs vs reward gate desynchronization (second aliasing source)
Observation latch (Run 016): _braking_received_latched = any_braking_peer_received (any peer ≤ –2.5).
Reward latch: _hazard_source_braking_latched = hazard_source_braking_received (only the injected target’s message).
They are separate variables in convoy_env.py. Even though they coincide 100 % in the current deterministic base_real scenario (sigma=0, speedDev=0), they are not guaranteed to be bit-for-bit identical on every single transition (mesh relay order, one-step lag, edge cases when intermediate peers relay first).
When ego[5] = 1.0 the critic sees “hazard”, but the reward terms (PENALTY_IGNORING_HAZARD / REWARD_EARLY_REACTION) may fire on a slightly different condition → the critic sees the same observation paired with inconsistent return targets.

One-step policy lag (aggravating factor)
Policy chooses action from obs_t–1. Reward is computed after the ESP-NOW update in step t. The exact transition where braking_received flips from 0 → 1 is the one where the model had already chosen “do nothing”. This adds noise exactly where the large reward swing occurs.

These three mechanisms together make the value target unlearnable. PPO mechanics are perfect (std monotonic drop, clip_fraction healthy, KL < 0.02) — the critic is simply receiving garbage supervision.
Why every previous hypothesis failed

“Just make the penalty bigger” — magnitude never mattered; the aliasing variance swamps it.
“HAZARD_PROBABILITY = 1.0” — eliminates the no-hazard case but leaves timing variance.
“Add binary feature” — makes the onset observable but does nothing for pre-onset aliasing or gate mismatch.
“Longer training” — critic cannot improve when the fundamental state-return mapping is non-Markovian.
